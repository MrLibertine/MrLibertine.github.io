Unity 基础速查

2019-06-07 806 字
Unity C#

MonoBehaviour 生命周期

Awake → OnEnable → Start → FixedUpdate → Update → LateUpdate → OnDisable → OnDestroy
  • Awake：脚本实例化时调用一次，适合初始化自身引用。此时其他脚本的 Awake 未必已执行。
  • OnEnable：每次对象激活时调用（含首次 Awake 后）。适合订阅事件。
  • Start：首次 Update 前调用一次。此时其他脚本的 Awake 已全部完成，可安全跨脚本引用。
  • FixedUpdate：固定时间步长调用（默认 0.02s），物理相关逻辑放这里。
  • Update：每帧调用，频率不固定。
  • LateUpdate：所有 Update 之后调用，适合跟随相机、处理动画后的逻辑。
  • OnDisable：对象禁用时调用，适合取消事件订阅。
  • OnDestroy：对象销毁时调用。

Update / FixedUpdate / LateUpdate 执行顺序

方法 调用时机 适用场景
FixedUpdate 固定间隔（Time.fixedDeltaTime） 物理运算、Rigidbody
Update 每帧 输入检测、通用逻辑
LateUpdate Update 之后 相机跟随、动画后期调整

要点：一帧内可能出现 0 次、1 次或多次 FixedUpdate（帧率 vs 物理步长差异）。物理逻辑放 FixedUpdate 避免偏移。

Transform 坐标系统

  • transform.position — 世界坐标
  • transform.localPosition — 相对父节点的局部坐标
  • TransformPoint(Vector3) — 局部 → 世界
  • InverseTransformPoint(Vector3) — 世界 → 局部
  • TransformDirection / InverseTransformDirection — 方向转换（忽略位移，仅旋转和缩放）

注意：当物体无父节点时，position 和 localPosition 等价。

Prefab 与 Instantiate

  • Prefab：可复用的 GameObject 模板，修改 Prefab 影响所有实例（未 Override 的属性）。
  • Prefab Variant：基于已有 Prefab 派生的子模板，继承基类属性并允许局部覆盖。
  • Override：实例上修改的属性会以粗体显示，可通过 Override 下拉菜单 Apply 回 Prefab 或 Revert。
  • Instantiate(Object original) — 克隆对象。返回值类型与传入类型一致。
  • Instantiate(Object original, Transform parent) — 克隆并设父节点，worldPositionStays 控制世界坐标保持。

协程原理

IEnumerator MyCoroutine()
{
    yield return new WaitForSeconds(1f);
    // 1 秒后继续执行
}
  • 本质是编译器生成的 IEnumerator 状态机，通过 MoveNext() 驱动。
  • yield return null — 等待一帧。
  • yield return new WaitForSeconds(n) — 等待 n 秒（受 Time.timeScale 影响）。
  • yield return new WaitForEndOfFrame — 等待到帧末渲染完成。
  • StartCoroutine 在 MonoBehaviour 上调用，对象禁用/销毁时协程自动停止。
  • 非 MonoBehaviour 类需注入 Mono 实例或用 MonoBehaviour 委托。

Destroy vs DestroyImmediate

  • Destroy(obj) — 当前帧结束时标记删除，下一帧才真正释放。
  • DestroyImmediate(obj) — 立即删除。仅编辑器脚本可用，运行时使用会破坏数据一致性（破坏顺序遍历等）。
  • Object.DontDestroyOnLoad — 场景切换时保持对象不被销毁。

GameObject / Component / Transform 关系

  • GameObject 是容器，持有 Component 列表，必然含一个 Transform。
  • Transform 是每个 GameObject 的强制组件，不可移除。
  • GetComponent<T>() 沿 GameObject 查找组件。
  • GetComponentInChildren<T>() 递归查找（含自身）。
  • GetComponentInParent<T>() 向上查找父级链。
  • TryGetComponent<T>(out T comp) — 无 GC 分配的获取方式（优于 null 检查 + GetComponent）。

常用时间相关 API

API 说明
Time.deltaTime 上一帧到当前帧的间隔（受 timeScale 影响）
Time.unscaledDeltaTime 不受 timeScale 影响的帧间隔
Time.fixedDeltaTime FixedUpdate 的固定时间步长
Time.time 游戏开始累计时间（受 timeScale 影响）
Time.timeScale 时间缩放，0 = 暂停
Time.realtimeSinceStartup 不受 timeScale 影响的真实运行时间
