Unity 渲染与性能速查

2019-06-07 965 字
Unity GLSL

DrawCall 与 SetPass Call

  • DrawCall：CPU 向 GPU 发起的一次绘制指令。每次 DrawCall 都有 CPU 开销（状态设置），是渲染性能的主要瓶颈之一。
  • SetPass Call：与 Shader Pass 绑定。一个 Shader 有多个 Pass 时，一次材质切换对应一个 SetPass。比 DrawCall 更昂贵。

SRP Batcher（URP/HDRP）：

  • 核心思路不是减少 DrawCall 数量，而是减少每次 DrawCall 的 GPU 状态设置开销。
  • 适用条件：Shader 必须兼容 SRP Batcher（使用 CBUFFER 声明的内置 UnityPerDraw 变量）。
  • 可以在 Frame Debugger 中查看是否被 Batcher 处理。

合批策略对照

合批方式 原理 适用场景 限制
Static Batching 运行时合并静态物体网格为一个大 Mesh 完全静止的物体（建筑、地面） 增加内存（额外存合批后的巨大 Mesh）
Dynamic Batching 每帧动态合并小网格 顶点数少的动态物体（< 300 顶点） 顶点/属性数限制、多 Pass Shader 无效
GPU Instancing GPU 用同一 Mesh+Material 绘制多个实例，仅变换矩阵不同 大量相同模型（草、子弹） 需要 Instancing 兼容的 Shader
SRP Batcher 缓存材质属性，减少 SetPass 开销 URP/HDRP 下的通用场景 需要 SRP Batcher 兼容 Shader

Overdraw（过度绘制）

  • 定义：同一像素被多次绘制，填充率浪费。
  • 检测：Scene View → 左上角下拉 → Overdraw 模式。越白越严重。
  • 优化策略：
    • UI 层减少重叠（空白的 RaycastTarget 关闭）。
    • 粒子特效缩小 Quad，用圆形 Mesh 代替方形。
    • 半透明物体排序靠后，先渲染不透明。
    • Forward 渲染路径下半透明无深度写入，叠加数层 Overdraw 严重。

LOD / MipMap / 剔除

技术 作用 配置位置
LOD (Level of Detail) 距离越远切换为低面数模型 LOD Group 组件
MipMap 距离越远使用低分辨率纹理 纹理导入设置 Generate Mip Maps
Frustum Culling 视锥体外物体不渲染 自动，无需配置
Occlusion Culling 被遮挡物体不渲染 Window → Rendering → Occlusion Culling，需烘焙

Canvas 渲染模式

模式 渲染方式 性能特点
Screen Space - Overlay 直接覆盖屏幕，无相机 最轻量，适合 HUD
Screen Space - Camera 绑定相机，可受相机后期影响 适合需要特效的 UI
World Space 作为 3D 空间物体，可旋转移动 适合血条、漂浮文字、VR UI

性能关键：Canvas 上任何子节点变化（位置、颜色、文本）都会触发 Canvas 重建（Rebuild），子节点多时开销大。用子 Canvas 隔离变化频繁的 UI 区域。

UI 合批条件

  • 同一 Canvas 下。
  • 使用同一材质（等于同一图集 + 同一 Shader）。
  • 渲染层级相邻（无不同材质的 UI 打断）。
  • 不再重叠的 UI 元素之间无 RaycastTarget 阻断合批（可把不需要点击的 Image 关闭 RaycastTarget）。

打断合批的因素：

  • 夹入不同图集的 Image。
  • 字体纹理与图集交替。
  • ScrollRect 内的 Mask 组件（每层 Mask 至少多 2 个 DrawCall）。

纹理压缩格式

平台 推荐格式 说明
iOS ASTC 高质量 + 任意尺寸，A 系列芯片支持
Android ASTC / ETC2 ASTC 中高端设备，ETC2 兼容全覆盖
Windows DXT5 (BC3) / BC7 BC7 质量最高但文件更大
WebGL DXT / ETC2 根据浏览器底层选择
  • 非 2 的幂纹理（NPOT）在部分平台无法压缩，统一用 POT 尺寸。
  • 带 Alpha 通道的纹理注意格式选择（ASTC 4×4 带 Alpha，ETC2 需 RGBA 扩展）。

Frame Debugger 速查

  • 路径：Window → Analysis → Frame Debugger → Enable。
  • 可以看到每一帧的 DrawCall 序列、SetPass 切换、Shader 属性变化。
  • 定位问题：某个 DrawCall 为什么没合批？它的材质、层级、Shader 是否与前一个相同？

常用性能指标参考（移动端）

指标 建议上限
DrawCall 200-400（视项目复杂度）
SetPass Call < 100
三角形数 < 200K（中低端）
纹理内存 < 200MB
帧率 30fps 稳定即可（高要求 60fps）
