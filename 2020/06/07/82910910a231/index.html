Unity 资源管理速查

2020-06-07 999 字
Unity C#

三种方案演进

Resources ──→ AssetBundle ──→ Addressables
  (小项目)      (手动管理)        (推荐方案)
方案 加载方式 热更新 内存管理 适用场景
Resources 同步/异步 ❌ 不可热更 手动 Unload 原型、小项目
AssetBundle AssetBundle.LoadFromFile 等 ✅ 手动 Unload(true/false) 传统手游
Addressables LoadAssetAsync<T> ✅ 引用计数自动释放 Unity 2018.3+ 推荐

Resources 目录

// 同步加载（阻塞主线程）
var prefab = Resources.Load<GameObject>("Prefabs/Enemy");

// 异步加载
var request = Resources.LoadAsync<GameObject>("Prefabs/Enemy");

// 加载所有同类型资源
var all = Resources.LoadAll<Texture>("Textures");

// 卸载未使用的资源
Resources.UnloadUnusedAssets();

注意事项：

  • Resources 目录下的所有资源都会打入包体，无论是否使用。
  • 同步 Load 会卡主线程，只在初始化阶段使用。
  • 资源不会被自动卸载，需手动 UnloadUnusedAssets 或场景切换释放。

AssetBundle 核心流程

构建

// Editor 脚本中构建
BuildPipeline.BuildAssetBundles(
    "Assets/StreamingAssets",
    BuildAssetBundleOptions.None,
    BuildTarget.Android
);

加载

// 从文件加载（最快，不占用额外内存）
var bundle = AssetBundle.LoadFromFile(path);

// 从内存加载（多一次拷贝，内存峰值高）
var bundle = AssetBundle.LoadFromMemory(bytes);

// 从网络流加载
var bundle = AssetBundle.LoadFromStream(stream);

// 加载资源
var prefab = bundle.LoadAsset<GameObject>("Enemy");
var allScenes = bundle.LoadAllAssets<Scene>();

// 异步
var request = bundle.LoadAssetAsync<GameObject>("Enemy");

卸载

bundle.Unload(false);  // 仅释放 Bundle 自身，已加载的资源保留
bundle.Unload(true);   // 释放 Bundle 及所有已加载的资源（危险）

关键陷阱：Unload(true) 后，已 Instantiate 的 GameObject 会丢失 Mesh/Texture/Shader 引用，变为紫色 Missing。确保无引用后再 Unload(true)。

AssetBundle 依赖管理

  • 多个 Bundle 依赖同一个资源（如贴图），需构建单独的依赖 Bundle。
  • 清单文件 AssetBundleManifest 记录依赖关系：
var manifestBundle = AssetBundle.LoadFromFile(manifestPath);
var manifest = manifestBundle.LoadAsset<AssetBundleManifest>("AssetBundleManifest");

// 加载目标 Bundle 前，先加载其所有依赖
string[] deps = manifest.GetAllDependencies("enemies");
foreach (var dep in deps)
    AssetBundle.LoadFromFile(depPath + dep);

// 再加载目标 Bundle
var enemyBundle = AssetBundle.LoadFromFile(bundlePath + "enemies");

注意：依赖 Bundle 也必须 Unload，且要在引用它们的 Bundle 之后卸载。

Addressables 核心操作

加载

// 异步加载（返回 handle）
var handle = Addressables.LoadAssetAsync<GameObject>("EnemyPrefab");
await handle.Task;
GameObject prefab = handle.Result;

// 实例化
var instanceHandle = Addressables.InstantiateAsync("EnemyPrefab");
await instanceHandle.Task;

// 加载场景
var sceneHandle = Addressables.LoadSceneAsync("Scenes/Battle", LoadSceneMode.Additive);

释放

// 释放单个资源
Addressables.Release(handle);

// 释放实例并销毁 GameObject
Addressables.ReleaseInstance(instanceHandle);

// 卸载场景
Addressables.UnloadSceneAsync(sceneHandle);

引用计数

Addressables 内部使用引用计数。每次 LoadAssetAsync 计数 +1，每个 Release 计数 -1。计数归零后资源自动释放。

常见 Bug：多次加载同一 Address 后只 Release 一次 → 资源永远不会释放 → 内存泄漏。确保每次 Load 都有对应 Release。

Group 策略设计

Group 内容 加载策略 释放策略
Local_Core 核心 Prefab、字体 启动时预加载 游戏退出才释放
Local_UI 通用 UI 面板 按需加载、缓存 场景切换时清理
Local_Level_N 关卡独有资源 进入关卡加载 退出关卡释放
Remote_Skins 皮肤、装饰资源 按需从 CDN 拉 LRU 缓存淘汰 
// 预加载 Group
Addressables.DownloadDependenciesAsync("Local_Core");
// 清理 Group 缓存（释放内存，不删磁盘）
Addressables.ClearDependencyCacheAsync("Local_Level_1");

热更新流程

  1. 构建时：Addressables 生成 catalog_xxx.hash + catalog_xxx.json + Bundle 文件。
  2. 启动时：对比本地 catalog hash 与远程 hash。
  3. 有更新：下载新 catalog → 根据 catalog 下载变化的 Bundle。
  4. 校验：CheckForCatalogUpdates → UpdateCatalogs → DownloadDependenciesAsync。
// 检查更新
var checkHandle = Addressables.CheckForCatalogUpdates(false);
await checkHandle.Task;
var updates = checkHandle.Result;

if (updates.Count > 0)
{
    // 更新 catalog
    var updateHandle = Addressables.UpdateCatalogs(updates, false);
    await updateHandle.Task;

    // 下载更新的资源
    var downloadHandle = Addressables.DownloadDependenciesAsync("Remote_Skins");
    await downloadHandle.Task;
}

内存红线速查

操作 内存影响
Resources.Load 加载到内存，永不自动释放
AssetBundle.LoadFromFile 仅加载文件头索引，用到时才读
AssetBundle.LoadFromMemory 完整加载到内存，多一次字节数组拷贝
Addressables.LoadAssetAsync 按需加载 + 引用计数
Resources.UnloadUnusedAssets 遍历全堆，主线程阻塞
AssetBundle.Unload(false) 释放索引，已加载资源留在内存
AssetBundle.Unload(true) 全部释放，破坏现有引用
